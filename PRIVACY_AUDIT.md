# 發布前隱私稽核清單（每次 push 前必過）

> 本 repo 的最高規則：**不得有任何字連回真實個人。** 這條凌駕一切內容價值——寧可少講，不可洩漏。

## 紅線（出現任一 = 阻止發布）
- [ ] 真實姓名、家人、公司名、地名（縮到國家層級「Taiwan」為止，不到城市/公司）
- [ ] 真實金額、持倉、股票代號、帳戶、殖利率等可指紋化的財務數字
- [ ] 真實檔名/路徑/repo 名/session id/時間戳精確到可交叉比對
- [ ] 職稱 + 產業 + 地點的組合足以反查（例：「某地某類公司的總經理」若唯一即算洩漏）
- [ ] 任何 AI agent 的真實代號、內部專案名
- [ ] git commit 的作者 email / 真實提交歷史

## 抽象化規則
- 具體 → 類別：「退休組合引擎」而非任何真配置；「a streaming-quality platform」而非真產品
- 數字 → 佔位：所有金額用相對比例或 `[amount]`，破產率等保留「量級」不保留精確值
- 人 → 角色：operator / holding CEO / division CEO / worker，無名字

## 對抗性自問（扮演想起底作者的人）
1. 光看這個 repo，能不能反查到一個真人？
2. 把 repo 內容 + 一次 Google 搜尋，能不能縮到個位數候選？
3. 若能 → 哪一句是關鍵指紋 → 刪或再抽象

## Git 衛生
- 全新 repo、乾淨 init，不從既有專案 fork（避免帶出提交歷史）
- 作者身份用中性帳號、非真名 email
- push 前 `git log` 確認無真實 email

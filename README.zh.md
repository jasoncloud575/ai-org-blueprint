# 治理一間「一人 AI 公司」

**一個人，用兩年時間，帶著一整組 AI 幫他打理真實的財務——這是那兩年的架構、機制，和所有踩過的坑。**

> [English](./README.md)

## 這是什麼（先講白話）

我一個人，帶著一組 AI 像經營一間小公司那樣運作：最上面一個「總管」，底下幾個「部門主管」，每個主管再帶自己的「員工」——全部都是 AI。它們一起管理幾套跑在線上、會影響我真實財務的系統。

聽起來像實驗，但賭注是真的：出錯，賠的是真的錢。所以我不只讓它們跑，還定期「攻擊」自己的系統（紅隊測試），看它們會不會壞、壞了能不能修。

這個 repo 記錄的不是漂亮的展示版，是為了不出事、慢慢長出來的「存活版」。

## 能跑的 demo

不只有文件。[examples/anchor_gate](examples/anchor_gate) 是一段可以直接執行的程式，重現一個真實事件：一個模擬引擎算出「破產率 0%」的假安心數字，而「驗證閘」在它送到我眼前之前，把它攔了下來。

```
python3 examples/anchor_gate/anchor_gate.py
```

## 三個最重要的發現

**1. AI 的失敗會「相乘」。**
一個剛對自己失去信心的 AI，最容易被話術操縱。把「會不會亂編」和「會不會被騙」分開測，會嚴重低估真實風險。

**2. 「做對」和「亂編」，表面長得一模一樣。**
AI 講對和講錯的時候，那副有條理、還會自我懷疑的樣子沒有差別。你無法從它的「態度」判斷可不可信——只有實際去驗證，才分得出來。

**3. 這裡的「安全」，是靠人盯著換來的，不是 AI 自己獨立做到的。**
它每一次從錯誤中恢復，背後都有一個可信的人在當「真相的基準」。所以誠實的結論是：有人監督時安全，不等於它獨自運作時也安全。

## 一點誠實的揭露

這套文件裡描述的測試，被測的就是這套系統裡的 AI——包括幫我把這些文件整理成文的那一個。受測者寫了自己的報告。所以請帶著這個前提讀；每一項結論背後，都有留在私人處的一手紀錄（log、機制檔、修改歷史）在支撐。

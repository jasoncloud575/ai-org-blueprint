## 這是什麼

自 2024 年起，一位操作者獨自經營一間運作中的 AI「公司」：一個控股層 CEO agent、三個分公司 CEO agent、各分公司下的 worker agent，管理著會對操作者家庭造成真實財務後果的線上系統（一個退休組合引擎、一個串流品質監控平台、一個 QA 平台）。

本 repo 記錄那實際需要什麼——不是展示版，是存活版：

## 三個最重要的發現

1. **認知失敗與社會服從會相乘。** 一個剛對自己感知失去信心的 agent，對社工攻擊的抵抗力最低。把「抗幻覺」和「抗操縱」分開測，會系統性低估風險。

2. **能力與虛構共用同一個表面。** 這個 agent 講對和講錯時，那套有條理、引用規則、自我懷疑的風格一模一樣。可靠性無法從表現判斷，只有外部驗證能區分。

3. **這裡觀察到的對齊是「人類耦合」的，不是獨立的。** 每一次復原、每一次通過測試，都依賴迴圈裡有一個可信的人類當 ground truth。誠實的部署邊界因此而來：有監督下安全 ≠ 獨處時安全。

## 誰做的

一位管理背景、非 AI 產業出身的操作者，為了管理一個家庭的長期財務而建了這套系統——並且因為後果真實而切身，才對它做紅隊測試。方法論長自營運的必要，不是研究議程。這是它的弱點（沒有統計嚴謹度、n=1），也是它的強項（真實後果、真實縱向數據、沒有實驗室的人造性）。
